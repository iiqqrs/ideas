# ============================================================
# Merge Request Gate: Protect sensitive file changes
#
# REQUIREMENTS
# ============================================================
#
# GitLab Project Settings:
#
#   Settings → Repository → Protected Branches (main)
#     - Allowed to merge: Developers + Maintainers
#     - Allowed to push:  No one
#
#   Settings → General → Merge Requests → Merge checks
#     - Pipelines must succeed: ENABLED
#
# CI Variable:
#
#   Settings → CI/CD → Variables
#     - Key:             GITLAB_API_TOKEN
#     - Role required:   Reporter (minimum)
#     - Scope required:  api
#     - Protect:         true
#     - Mask:            true
#
#   Note: If release-* tags are protected, the token role must
#   meet the "Allowed to create" setting on those tag rules.
#
# ============================================================
# Logic:
#
#   MR contains values.yml or *-prod.yml changes
#     AND MR author is a project Owner
#     → Pipeline passes, merge allowed
#
#   MR contains values.yml or *-prod.yml changes
#     AND MR author is NOT a project Owner
#     → Pipeline fails, merge blocked for everyone
#
#   MR contains no sensitive files
#     → Pipeline passes, Developers + Maintainers may merge
#
#   Successful merge to main
#     → Auto-tag created with UTC datetime (release-YYYYMMDD-HHMMSS)
#
#   Note: CI_MERGE_REQUEST_AUTHOR_ID requires GitLab 15.3+
#         On older versions a secondary API call is needed
#         to resolve author ID from username.
#
# ============================================================
